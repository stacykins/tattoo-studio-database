# Лабораторна робота №1: Збір вимог та розробка схеми ER

## 1. Короткий виклад вимог
**Призначення системи:** Автоматизація тату-студії для обліку сеансів, майстрів та розхідних матеріалів.

### Потреби зацікавлених сторін:
- **Адміністратор:** Моніторинг завантаженості майстрів та залишків на складі.
- **Майстер:** Перегляд розкладу та деталей ескізу (місце на тілі, складність).
- **Клієнт:** Збереження історії татуювань та медичних особливостей (алергії).

### Бізнес-правила:
- Клієнт повинен бути старше 18 років.
- Один сеанс проводиться одним майстром.
- Ціна сеансу фіксується в момент запису.

## 3. Список сутностей та атрибутів

### Artists (Майстри)
- `artist_id` (PK) — ID майстра
- `name` — ПІБ
- `specialization` — Стиль (Old School, Realism тощо)

### Clients (Клієнти)
- `client_id` (PK) — ID клієнта
- `phone` — Номер телефону
- `medical_info` — Алергії/протипоказання

### Sessions (Сеанси)
- `session_id` (PK) — ID сеансу
- `artist_id` (FK) — Посилання на майстра
- `client_id` (FK) — Посилання на клієнта
- `session_date` — Дата та час
- `price` — Вартість

---

## 4. Припущення
- Вважаємо, що один сеанс = одне татуювання.
- Матеріали списуються за шаблоном (наприклад, 1 сеанс = 2 рукавички, 1 голка).
